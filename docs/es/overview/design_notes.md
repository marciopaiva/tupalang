# Notas de Diseño (IA)

## Propósito

Registrar lineamientos de producto, comunidad y calidad para orientar decisiones de diseño.

## Producto

- Mantener un núcleo estable y documentado.
- Evitar funcionalidades grandes antes del MVP.
- Declarar claramente lo que no está listo.

## Comunidad

- Centralizar soporte en GitHub Discussions.
- Mantener el FAQ corto y objetivo.
- Publicar RFCs para decisiones de diseño.

## Calidad

- Pruebas de regresión para parsing y tipos.
- Linters y formateadores con reglas mínimas.
- Mensajes de error con ejemplos de corrección.
- Salida JSON para integración con herramientas y automatizaciones.

## Documentación

- Guía de inicio rápido.
- Glosario de términos.
- Comparaciones honestas con otros lenguajes.

## RFCs

- Usar issues con el prefijo `[RFC]` para propuestas de diseño.
- Consolidar decisiones aprobadas en la SPEC y en la documentación relevante.

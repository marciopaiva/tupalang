# Guía de Gobernanza

## Propósito

Definir roles, proceso de decisión y uso de RFCs.

## Roles

- **Mantenedores**: aprueban cambios, revisan PRs y definen la hoja de ruta.
- **Colaboradores**: proponen mejoras vía issues y PRs.

## Proceso de decisión

- Abrir discusiones en issues/PRs.
- Decisiones registradas en docs y/o SPEC.
- Cambios grandes requieren un RFC.

## RFCs

- Abre una issue con el prefijo `[RFC]`.
- Incluye motivación, alternativas e impacto.
- Tras consenso, actualiza la SPEC y los docs relevantes.

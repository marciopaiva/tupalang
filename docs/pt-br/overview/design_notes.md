# Notas de Design (IA)

## Propósito

Registrar diretrizes de produto, comunidade e qualidade para orientar decisões de design.

## Produto

- Manter um núcleo estável e documentado.
- Evitar funcionalidades grandes antes do MVP.
- Declarar claramente o que não está pronto.

## Comunidade

- Centralizar suporte em GitHub Discussions.
- Manter o FAQ curto e objetivo.
- Publicar RFCs para decisões de design.

## Qualidade

- Testes de regressão para parsing e tipos.
- Linters e formatadores com regras mínimas.
- Mensagens de erro com exemplos de correção.
- Saída JSON para integração com ferramentas e automações.

## Documentação

- Guia de início rápido.
- Glossário de termos.
- Comparações honestas com outras linguagens.

## RFCs

- Usar issues com o prefixo `[RFC]` para propostas de design.
- Consolidar decisões aprovadas na SPEC e na documentação relevante.
